# ESTune
To enhance the efficiency of existing iterative tuning methods, we propose ESTune. It implements an early stopping strategy for underperforming configurations, thereby eliminating unnecessary time consumption and enabling more iterative exploration within a fixed time frame. The early stopping strategy in ESTune is primarily implemented through HBNN, MAML, and an evaluator. Since the evaluator is merely a simple value-judgment process, the relevant code is omitted here. Therefore, we provide detailed implementations of the HBNN and MAML modules. 

It is important to note that the decision not to provide the complete code integrated with other tuning algorithms is primarily based on the following two considerations: (1) numerous publicly available implementations of tuning methods already exist, making it unnecessary to replicate others' work; (2) providing the entire code could increase the reader's cognitive load, thus making the understanding of the ESTune implementation more challenging.
